# Config (Layer 3 Reference Material)

This folder contains stable reference material that persists across all pipeline runs. Stage CONTEXT.md files declare which of these files they load.

| File | Purpose | Loaded by |
|------|---------|-----------|
| `definition-of-done.md` | Project-wide DoD checklist | Stages 3, 4 |
| `naming-conventions.md` | Branch, file, commit naming rules | Stages 2, 3 |
| `tech-stack.md` | Approved technologies and versions | Stage 2 |
| `coding-standards.md` | Code style, linting, review checklist | Stage 2 |

> These files are configured once per project. Update them when the project's standards change, not per-requirement.
