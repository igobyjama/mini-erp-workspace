# Feature Documentation: req-001

## Overview

This document covers the project foundation for mini-erp: the pnpm monorepo structure, Fastify API backend with JWT authentication, React frontend scaffold, PostgreSQL via Prisma, Docker Compose development environment, and GitHub Actions CI pipeline. All subsequent modules (req-002 onwards) are built on top of this foundation.

---

## API Changes

### `GET /api/v1/health`

**Description:** Liveness check. No authentication required. Used by Docker health checks and uptime monitors.

**Response:**
```json
{ "status": "ok" }
```

**Error codes:** None — always returns 200 if the process is running.

---

### `POST /api/v1/auth/login`

**Description:** Authenticates a user with email and password. On success, sets an httpOnly JWT cookie.

**Request:**
```json
{
  "email": "string — valid email address",
  "password": "string — plain-text password (transmitted over HTTPS only)"
}
```

**Response (200):**
```json
{ "ok": true }
```
Cookie set: `token=<jwt>; HttpOnly; SameSite=Strict; Path=/; Max-Age=86400`

**Error codes:**
| Code | Body | Meaning |
|------|------|---------|
| 400 | `{ "error": "Invalid request body" }` | Email or password field invalid/missing |
| 401 | `{ "error": "Invalid credentials" }` | Wrong password or unknown email |

---

### `POST /api/v1/auth/logout`

**Description:** Clears the JWT cookie. No authentication required.

**Response (200):**
```json
{ "ok": true }
```
Cookie cleared: `token=; Path=/; Max-Age=0`

---

### Protected routes

All routes registered under `/api/v1/` (except `/health`, `/auth/login`, `/auth/logout`) require a valid `token` cookie.

**Error when missing or invalid (401):**
```json
{ "error": "Unauthorized" }
```

## Configuration Changes

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `DATABASE_URL` | string | — | PostgreSQL connection string (required) |
| `API_PORT` | number | `3000` | Port the Fastify server listens on |
| `API_HOST` | string | `localhost` | Bind address (`0.0.0.0` in Docker) |
| `JWT_SECRET` | string | — | HMAC-HS256 signing secret, min 32 chars (required) |
| `JWT_EXPIRES_IN` | string | `24h` | JWT expiry (e.g. `24h`, `7d`) |
| `SEED_USER_EMAIL` | string | — | Initial user email for seed script |
| `SEED_USER_PASSWORD` | string | — | Initial user password for seed script |

All variables are documented in `.env.example` at the repository root.

## Database Changes

### New table: `users`

| Column | Type | Constraints |
|--------|------|-------------|
| `id` | `uuid` | PK, default `gen_random_uuid()` |
| `email` | `varchar` | unique, not null |
| `password_hash` | `varchar` | not null |
| `created_at` | `timestamptz` | default `now()` |
| `updated_at` | `timestamptz` | auto-updated |

Migration file generated by `prisma migrate dev --name init`.

## Component / Module Changes

### `packages/shared`

Exports shared Zod schemas and TypeScript types used by both the API and the web app.

| Export | Description |
|--------|-------------|
| `loginSchema` | Zod schema: `{ email: string (email), password: string (min 1) }` |
| `LoginInput` | TypeScript type inferred from `loginSchema` |

### `apps/web` — Login page (`/login`)

- React Hook Form with `loginSchema` (Zod) resolver
- Calls `POST /api/v1/auth/login` via TanStack Query mutation
- On success: navigates to `/`
- On error: displays error message inline

### `apps/web` — Dashboard (`/`)

- Route guard: attempts auth check before load; redirects to `/login` if unauthenticated
- Stub content — modules added in req-002+

## Usage Examples

### First-time setup

```bash
# 1. Copy env file and fill in values
cp .env.example .env

# 2. Install dependencies
pnpm install

# 3. Generate Prisma client
pnpm prisma:generate

# 4. Run migrations (requires DATABASE_URL in .env)
pnpm prisma:migrate

# 5. Seed initial user
pnpm prisma:seed

# 6. Start dev servers
pnpm dev
# API → http://localhost:3000
# Web → http://localhost:5173
```

### Docker Compose (full stack)

```bash
cp .env.example .env
# Edit .env — set JWT_SECRET, SEED_USER_EMAIL, SEED_USER_PASSWORD

docker compose -f docker/compose.dev.yml up
# API → http://localhost:3000
# Web → http://localhost:5173
```

### Running tests

```bash
pnpm test
```

### Login via curl

```bash
curl -c cookies.txt -X POST http://localhost:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","password":"yourpassword"}'
# → { "ok": true }

# Use the cookie on a protected endpoint
curl -b cookies.txt http://localhost:3000/api/v1/health
```
